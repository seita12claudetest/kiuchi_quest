# 受入仕様

## 縦切り完成条件

最初のリリース候補は「会社を退勤し、駅前を歩き、横丁の店へ入り、会話・注文・戦闘を経て帰宅する」15〜20分の体験とする。

### 移動

- キーを3秒保持しても設計速度を±5%以内で維持する。
- 30/60/120fpsで移動距離が同一になる。
- 壁、机、NPCを通過しない。
- 会話後にキーが入りっぱなしにならない。
- ゲームパッドとタッチでも同じ入口へ到達できる。

### 接続

- 会社、駅前、横丁、5店舗、自宅の全出入口が双方向に通じる。
- 外観の扉と当たり判定が視覚的に一致する。
- 遷移後に壁内、画面外、入口再発火位置へ出ない。
- 閉店中は理由と営業時間を表示する。
- 自動接続テストに未定義・片方向ワープがない。

### 相互作用

- 画面内の主要NPC全員と会話できる。
- 各店舗に調査対象が3個以上ある。
- 近接対象と操作ボタンをプロンプト表示する。
- 会話、注文、クエストが好感度・所持金・時間へ反映される。

### 戦闘

- 横丁までに任意通常戦を最低1回、回避可能な形で配置する。
- 木内＋仲間最大4人でコマンド戦闘できる。
- 攻撃、仕事術、人脈、アイテム、交渉、作戦、逃走が機能する。
- 勝利、交渉解決、逃走、敗北に別の現実結果がある。
- 通常戦は中央値90秒以内で終わる。

### 育成

- 一日の選択結果が能力、健康、好感度、所持金、時間に残る。
- 同じ開始状態から異なる3行動で異なる結果になる。
- 結果画面から「なぜ変化したか」を追跡できる。

### 公開

- Godot Web exportがGitHub Pagesで起動する。
- 初回ロード後に致命的コンソールエラーがない。
- macOS / Windows export用プリセットを保持する。
- セーブ後の再読込で位置、時刻、仲間、クエスト、育成値が一致する。

## 憲法検査

各機能PRは以下に回答する。

1. この変更は育成・発見・自分の経験のどれを強めるか。
2. プレイヤーが次に何をすればよいか理解できるか。
3. マップ上の見えるものと実際の機能が一致するか。
4. 失敗が理不尽な停止ではなく次の展開になるか。
5. 参考作品の固有表現を複製していないか。
