# 世界・マップ接続仕様

## 接続原則

- 外観の扉座標と室内の出口座標をデータで双方向に結ぶ。
- 出入口には名称、方向、遷移先、出現位置、営業時間、解放条件を持たせる。
- 入口前では「入る: ○○」を表示する。
- 閉店中は入れない理由と次の開店時刻を表示する。
- 遷移後は出口を背にして立ち、即座に再遷移しない安全距離を置く。
- すべての接続を自動検証し、片方向リンク、壁内出現、未定義マップをCIで失敗させる。

## 店舗必須データ

- `id`, 表示名、外観マップ、室内シーン
- 入口・出口ペア
- 営業曜日と時刻
- 店主と常連客
- 商品または料理、価格、在庫条件
- 初回会話、通常会話、好感度会話、年代会話
- 受注・進行・完了可能なクエスト
- 調べられる物を最低3個
- 年代差分（看板、商品、価格、人物、閉店・改装）

## マップ上の行動

- 話す: 正面1マスのNPC
- 調べる: 看板、机、端末、棚、電話、掲示物、落とし物
- 入る: 扉、階段、改札
- 買う・注文: カウンターまたは店主
- 仕事: 担当設備、書類、会議、顧客
- 拾う: 明示されたアイテム
- 仲間に誘う: 関係・予定・場所の条件を満たすNPC

## 移動

- 通常歩行: 96 px/sを初期値とし実機試験で調整
- ダッシュ: 156 px/s。屋内と混雑時は制限可能
- 4方向移動、入力方向へ向いてから移動
- 加速0.08秒、減速0.10秒
- 斜め入力時の速度正規化
- 壁沿い補正を弱く設け、入口へ入りやすくする
- 会話・プロンプト範囲では自動減速
- キーリピートやフレーム数ではなくdelta時間で制御
