# Godot移行・実装計画

## 方針

既存Web版を即座に削除しない。Godot版が縦切り受入条件を満たすまで現行版を公開維持し、データと仕様を移植する。Godot MCPは高リスク登録のため、現段階では導入せず、エディタとCLIの標準機能を使用する。

## Phase 0: 仕様確定

- 制作憲法、全体仕様、世界接続、戦闘、受入仕様をレビュー
- 仮の店名、仲間候補、時刻単位を確定
- 著作権・商標・素材ライセンス境界を確認

終了条件: 未決定事項に担当と期限があり、縦切りの受入条件が承認されている。

## Phase 1: Godot基盤

- Godot 4プロジェクト、Web/macOS/Windows export preset
- 入力マップ、解像度、ピクセル整数倍、セーブ基盤
- CharacterBody2Dによるdelta時間移動
- TileMapLayer、当たり判定、カメラ、相互作用検出
- 自動テスト可能なデータ形式

終了条件: テストマップで速度、衝突、会話プロンプトが受入基準を満たす。

## Phase 2: 接続された世界

- 会社、駅前、横丁、木内宅
- 横丁5店舗の外観・室内
- 出入口レジストリと接続検査
- 営業時間と閉店表示

終了条件: 全地点を歩いて往復でき、見える扉と遷移が一致する。

## Phase 3: 暮らしと育成

- 時刻・曜日・勤務義務
- 仕事、会話、注文、買い物、帰宅
- 能力、健康、好感度、所持金、ログ
- パワプロ型の行動予測と結果表示

終了条件: 一日を3通りに過ごし、異なる木内が育つ。

## Phase 4: 通常戦闘と仲間

- 心象世界への遷移
- 最大4人のターン制戦闘
- 7コマンド、敵意図、適性、状態
- 仲間加入、編成、連携技
- 勝利・交渉・逃走・敗北の現実結果

終了条件: 通常敵とボスを複数の解法で攻略できる。

## Phase 5: 縦切り完成

- 1988年初週のイベントとAS/400題材
- 店主、同僚、敵、料理、クエストを本番内容へ置換
- 音、演出、アクセシビリティ、タッチ操作
- Web実機QAとセーブ互換性

終了条件: `acceptance-tests.md`の全項目を満たす。

## Phase 6: 年代・キャリア拡張

- 月次、年度、昇進、異動
- 1988年から現代までの技術・社会イベント
- 店舗と街並みの年代差分
- 主任、係長、課長代理、課長、副部長の責任変化

## 移植対象

- 会社史・JTC調査
- 食事と健康データ
- キャリア階段と基礎ステータス
- ライセンス確認済み音素材
- 木内キャラクター案

## 作り直す対象

- Canvas描画、入力、マップ、ワープ、衝突
- 背景画像と無関係な店舗座標
- 汎用文の大量NPC・クエスト・イベント
- パネルボタンによる瞬間移動中心の行動
- フレーム依存移動
