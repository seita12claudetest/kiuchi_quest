# 実装計画

## 目的

「会社員の日常」「健康診断の数値」「昭和横丁の飲み会」をRPGの成長・戦闘・収集要素として遊べる形にする。

単に健康数値を下げるだけでなく、飲み会を極めるプレイ、生活を改善するプレイ、図鑑やクエストを埋めるプレイのいずれにも到達点を用意する。

## Phase 1：初期プレイアブル版（実装済み）

- ブラウザのみで動く静的Webゲーム
- マップ移動、施設マス、ランダムイベント
- 曜日・時間帯システム
- ターン制バトルと必殺技
- レベル・経験値・アイテム・ショップ
- 健康指標と病院イベント
- クエスト、図鑑、好感度、称号、実績
- 3枠セーブとAUTOセーブ
- 複数エンディングと隠しボス
- PC・スマートフォン操作

## Phase 2：コンテンツ拡張

### イベント

- 月曜朝、金曜夜、給料日、健康診断前日など曜日・時間限定イベント
- 上司、同僚、医師、店主、トレーナーごとの会話分岐
- 血圧・尿酸・コレステロールの組み合わせによる特殊イベント
- 低確率の隠しイベントと周回後イベント

### 戦闘・育成

- 状態異常（泥酔、二日酔い、空腹、ストレス）
- 装備品とパッシブスキル
- 敵の行動パターンとボス専用演出
- レベルアップ時の能力選択
- 難易度と報酬のバランス調整

### 収集・周回

- アイテム・敵・イベント図鑑の説明文追加
- エンディング回収率表示
- 周回引き継ぎ
- 追加称号・実績

## Phase 3：UI・演出強化

- マップ・キャラクター・敵のドット絵差し替え
- 攻撃、回復、レベルアップ、緊急搬送のエフェクト
- BGM・SEの音量設定とミュート
- 会話ウィンドウ、イベントカットイン
- PWA対応とホーム画面追加
- キーボード、タッチ、アクセシビリティ改善

## Phase 4：品質向上・公開

- ゲーム状態とエンディング条件の自動テスト
- セーブデータのバージョン移行テスト
- Chrome、Edge、Safari、スマートフォンでの動作確認
- 難易度、テンポ、数値バランスのプレイテスト
- GitHub Pagesでの公開

## 優先順位

1. 既存機能の不具合修正と遊びやすさ
2. イベント・会話・敵の追加
3. エンディング到達条件とバランス調整
4. ドット絵、エフェクト、サウンド
5. 周回・追加コンテンツ

## 完了条件

- 初見で説明を読まずとも移動・戦闘・回復ができる
- 30分程度のプレイで少なくとも1つのエンディングに到達できる
- 健康重視と飲み会重視で明確に異なる結果になる
- セーブ・ロード後もクエスト、図鑑、好感度、実績が保持される
- PCとスマートフォンの両方で最後まで操作できる
